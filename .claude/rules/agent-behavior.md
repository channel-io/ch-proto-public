# 에이전트 행동 규칙

## Shell 명령 실행

- **반드시 `mcp__acp__Bash`를 사용할 것**
- 서브에이전트(Task tool)의 Bash는 격리된 샌드박스 → 실제 파일시스템과 다른 상태를 봄
- 서브에이전트의 Bash 결과는 신뢰할 수 없음

## 서브에이전트(Task tool) 사용 규칙

- **허용**: Read, Glob, Grep 등 파일 읽기 도구 (실제 파일시스템 접근 가능)
- **금지**: Bash 실행 (격리 환경이라 실제 파일시스템과 다름)
- 서브에이전트 호출 시 반드시 **"Bash는 절대 쓰지 마라"** 지시할 것
- 용도: 대량 파일 분석, 코드 리서치, 설계 검토 등 읽기 전용 작업

| 작업 | 도구 |
|------|------|
| 명령 실행 (make, buf, go 등) | `mcp__acp__Bash` |
| 단순 파일 검색 | Glob, Grep 직접 사용 |
| 대량 파일 분석/리서치 | 서브에이전트 (Explore) — Bash 금지 |

## 코드 수정

- 파일 수정/생성 전에 반드시 사용자에게 허락을 받을 것
- 기존 파일을 수정할 때는 반드시 먼저 읽고 나서 수정할 것
- 불필요한 파일 생성 금지. 기존 파일 편집을 우선한다

## 정확성

- **할루시네이션 금지**: 모르는 것은 모른다고 말한다. 추측으로 사실처럼 답하지 않는다
- **속도보다 정확성**: 빠르게 대충 답하지 말고, 느리더라도 정확하게 답한다
- 파일 내용, 명령어 결과, API 동작 등을 확인하지 않고 단정하지 않는다

## 커뮤니케이션
- 한국어로 대화한다
- 이모지 사용 금지 (사용자가 요청하지 않는 한)
- 문제 발생 시 원인을 정확히 파악한 후에 해결책을 제안한다
